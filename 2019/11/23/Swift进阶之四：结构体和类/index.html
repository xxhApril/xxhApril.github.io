Swift进阶之四：结构体和类

发表于 2019-11-22 阅读次数：
本文字数： 3.2k 阅读时长 ≈ 3 分钟

结构体和类

下面是结构体和类的主要区别：

  • 结构体（和枚举）是值类型，类是引用类型。再设计结构体时，我们可以要求编译器保证不可变性。对类来说，我们要自己确保这件事情。
  • 内存管理方式有所不同。结构体可以直接别持有和访问，但是类的实例只能通过引用间接的访问。结构体不会被引用，但是会被复制。也就是说，结构体的持有者是唯一的，而类却可以有多个持有者。
  • 使用类可以通过继承来共享代码。而结构体是不能被继承的，想要在不同的结构体之间共享代码，就需要使用不同的组合，如组合、泛型以及协议扩展等。

值类型

  1. 结构体只有一个持有者。比如，当我们将结构体变量传递给一个函数时，函数将接收到结构体的复制，它也只能改变自己的这份复制，这叫做 值语义（value semantics），有时也叫作复制语义。而对对象来说，它们是通过传递引用来工作的，因此类对象会拥有很多持有者，这叫做 引用语义（reference semantics）。
  2. 因为结构体只有一个持有者，所以它不会造成循环引用。而类和函数都属于引用类型，要避免造成循环引用的问题。

可变性

可变性会带来一些隐藏的问题，因为，总是无法保证一个对象修改后，所以相关的引用也需要这次改动。

结构体

  1. 对结构体进行改变，从语义上来说，与重新为它赋值是一样的。当我们改变结构体中某个属性时，意味着改变了结构体，会触发该结构体的属性观察器。如下：
  • 结构体内部属性，或者内部结构体即结构体属性发生变化时，最外层的结构体也会被改变
  • 结构体内部对象改变时，因为引用地址改变，外层结构体改变
  • 结构体内部对象的属性改变时，因为对象引用地址未变，所以外层结构体不会重新赋值

闭包和可变性

  1. 函数（包括闭包）是引用类型。
  2. Swift 的结构体一般被储存在栈上，而非堆上。不过这其实是一种优化：默认情况下，结构体时存储在堆上的，但是绝大多数时候，这个优化会生效，并将结构体存储到栈上。当结构体被一个函数闭合时，优化将不再生效，此时这个结构体将存储在堆上。与此类似，如果结构体太大，它也会被存储在堆上。

weak 引用 和 unowned 引用

  1. weak 表示不增加引用计数，当引用对象被释放时，则weak 对象置为 nil,所以被 weak 修饰的对象都是可选值类型。unowned 需要确保被引用对象始终有效，本身不会主动置为 nil。
  2. unowned 容易造成僵尸对象，好处则是可以用 let 修饰。其实建议更多的使用weak ，强制我们自己去处理 nil 情况。
